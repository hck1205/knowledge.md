# 협업 운영

## 협업 도구

- ClickUp을 사용해서 업무 트래킹과 협업 진행

## ClickUp에 관리하면 좋은 항목

- 기획
- 모바일 개발
- 서버 개발
- API 연동
- QA
- 아이디어 백로그

## 추천 운영 방식

### 1. 큰 단위보다 작은 작업으로 쪼개기

예시:

- 로그인 화면 초안 만들기
- 토픽 조회 API 명세 정리
- 투표 결과 UI 연결하기

### 2. 서버-모바일 연결 포인트를 명확히 남기기

- 요청/응답 형식
- 에러 케이스
- 필수/선택 필드
- 인증 방식

### 3. 결정 사항은 문서에도 남기기

ClickUp은 업무 진행을 추적하기 좋고, 문서는 왜 그렇게 정했는지 맥락을 남기기 좋다.  
두 도구를 같이 써야 나중에 다시 봐도 이해하기 쉽다.

## 추천 태스크 분류

- `Backlog`
- `Todo`
- `In Progress`
- `Review`
- `Done`

## 협업 메모

- 서버와 모바일은 서로 막히기 쉬운 지점이 로그인과 API 명세다
- 따라서 초반에는 화면 구현과 API 계약 정리를 병행하는 것이 효율적이다
- 앱이 단순한 만큼, 오히려 커뮤니케이션 누락이 생기지 않게 관리하는 것이 중요하다

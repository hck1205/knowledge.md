# MVP 범위

## MVP에서 꼭 필요한 기능

- 토픽 1개 보기
- 두 선택지 중 하나 선택하기
- 투표 제출하기
- 투표 결과 보기
- 다음 토픽으로 이동하기
- 로그인한 사용자 기준으로 투표 이력 처리하기

## Android 앱 기준 화면 범위

- 스플래시 또는 앱 진입 화면
- 로그인 화면
- 메인 투표 화면
- 투표 결과 화면 또는 결과 영역

## 초기 버전에서 굳이 없어도 되는 것

- 댓글 기능
- 토픽 직접 생성 기능
- 랭킹 시스템
- 친구 초대 기능
- 복잡한 프로필 기능
- 알림 고도화

## 모바일 구현 관점 메모

- 가장 중요한 화면은 메인 투표 화면이다
- 사용자는 생각보다 오래 읽지 않으므로, 질문과 선택지가 즉시 눈에 들어와야 한다
- 결과는 숫자와 비율이 직관적으로 보이면 충분하다
- 첫 버전에서는 UI를 화려하게 만들기보다 흐름이 끊기지 않는 것이 더 중요하다

## 서버 의존 사항

- 로그인 API
- 토픽 조회 API
- 투표 제출 API
- 결과 조회 API

## MVP 성공 기준

- 로그인 후 토픽을 보고 투표할 수 있다
- 투표 직후 결과가 정상적으로 보인다
- 여러 개 토픽을 무리 없이 넘겨볼 수 있다
- Android 앱과 서버 간 기본 플로우가 안정적으로 연결된다

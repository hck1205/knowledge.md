# 노트 패턴

모든 노트를 같은 템플릿에 억지로 맞추지 않는다.
노트의 역할에 따라 가장 어울리는 형태를 고른다.

## 상시 지식 노트

시간이 지나도 계속 참고할 수 있는 개념에 사용한다.

권장 구조:

1. 개념명 또는 질문형 제목
2. 짧은 정의 또는 핵심 주장
3. 주요 포인트
4. 연결할 관련 노트
5. 필요하면 예시나 적용 맥락

잘 어울리는 주제:

- 소프트웨어 엔지니어링 개념
- 멘탈 모델
- 정의 정리
- 반복해서 떠올릴 교훈

## 개인 철학 노트

가치관, 신념, 확신, 갈등, 성찰을 담는 데 사용한다.

권장 구조:

1. 1인칭 진술 또는 질문
2. 왜 중요한지
3. 긴장점, 예외, 흔들리는 지점
4. 관련 경험, 영향, 연결 노트

교과서 같은 어조보다 사유의 목소리를 우선한다.
사용자 의도가 그렇다면 미완성 생각도 미완성 상태로 남겨둔다.

## 학습 노트

개발 공부, 이직 준비, 영어 공부에 사용한다.

권장 구조:

1. 주제
2. 이게 무엇인지
3. 예시
4. 자주 헷갈리는 점이나 실수
5. 연습 질문 또는 다음 복습 포인트

기술 학습은 정확성과 실행 가능한 예시를 우선한다.
영어 학습은 짧은 예문, 교정 포인트, 재사용할 표현을 넣는다.

## 프로젝트 문서

현재 진행 중인 일의 실행 맥락을 남길 때 사용한다.

권장 섹션:

1. 목표
2. 현재 상태
3. 내린 결정
4. 열려 있는 질문
5. 다음 액션

시간 흐름이 중요한 문서라면 과거 내용을 덮어쓰기보다 날짜 로그를 이어 붙인다.

## 아이디어 노트

아직 다듬어지지 않은 초기 생각을 담을 때 사용한다.

권장 구조:

1. 한 줄 아이디어
2. 왜 흥미로운지
3. 더 파볼 거리
4. 다음 작은 행동

마찰을 낮게 유지한다.
완성도보다 포착을 우선한다.

## 참고 노트

원래 생각을 전개하기보다는 보조 자료를 모아둘 때 사용한다.

잘 어울리는 내용:

- 링크 모음
- 읽은 자료 정리
- 체크리스트
- 코멘트가 붙은 발췌문

나중에 봐도 이해되도록 최소한의 맥락은 꼭 남긴다.
